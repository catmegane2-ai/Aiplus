# 開発運用メモ（Git / ワークフロー）

## Git 運用方針

* 編集は基本的に **ローカルで行う**
* GitHub 上で直接編集しない（履歴ズレ防止）
* 作業開始時は必ず `git pull` で最新化
* コミットは **1タスク = 1コミット**
* Push が拒否された場合は履歴衝突が原因。無理に続行しない
* AI アシストはフローを止めず、実行可能な形で提示する

## 想定する日常フロー

1. `git pull` で同期
2. 編集（1タスクに限定）
3. `git add .`
4. `git commit -m "..."`
5. `git push`

## 注意点

* リモートに直接変更を入れない
* ローカルとリモートが食い違ったら、まず状況確認
* 必要なら `git pull --rebase` を使うが、常用しない
